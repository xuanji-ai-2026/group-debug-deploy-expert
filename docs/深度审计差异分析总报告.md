## ⚠️ 发现的问题与风险 (Issues & Risks)

### 🔴 问题 P1 (Critical): 无法实时验证服务器一致性

**问题描述**:
当前环境缺少SSH客户端，无法直接连接远程服务器 (43.160.237.122) 进行实时文件对比和状态检查。

**影响评估**:
- ❌ 无法确认远程服务器是否运行最新代码
- ❌ 无法验证配置文件是否一致
- ❌ 无法检查服务运行状态和错误日志
- ❌ 存在生产事故风险 (如配置漂移导致异常)

**修复方案** (按优先级排序):

#### 方案A: 启用Windows OpenSSH客户端 (推荐, 5分钟)

```powershell
# 以管理员身份运行PowerShell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0

# 验证安装
ssh -V

# 测试连接
ssh -i "D:\BeijiXing-AI\singapore.pem" root@43.160.237.122 "echo '连接成功'"
```

#### 方案B: 安装Git for Windows (包含SSH, 10分钟)

```bash
# 下载安装: https://git-scm.com/download/win
# 安装时确保勾选 "Git Bash Here" 和 "Use Git from Windows Command Prompt"

# 使用Git Bash执行SSH命令
ssh -i "/d/BeijiXing-AI/singapore.pem" root@43.160.237.122
```

#### 方案C: 使用WSL2 (最完整, 30分钟)

```bash
# 启用WSL2
wsl --install

# 在WSL中安装OpenSSH-client (通常已预装)
sudo apt update && sudo apt install openssh-client -y

# 设置密钥权限
chmod 400 /mnt/d/BeijiXing-AI/singapore.pem

# 连接服务器
ssh -i /mnt/d/BeijiXing-AI/singapore.pem root@43.160.237.122
```

**建议**: 立即执行方案A，完成后运行校验脚本 `check-server-consistency.ps1`

### 🟡 问题 P2 (High): 前端依赖文件可能被误统计

**现象观察**:
- web-admin: **6033个 JS文件** ← 异常偏高
- web-pc: **4045个 JS文件** ← 异常偏高

**根因分析**:
这很可能是包含了 `node_modules/` 目录下的第三方依赖文件，而非项目源码。

**验证方法**:

```powershell
# 检查实际的源代码文件数 (排除node_modules)
$webAdminSrc = (Get-ChildItem "D:\BeijiXing-AI\frontend\web-admin\src" -Recurse -Include *.js,*.vue).Count
$webPcSrc = (Get-ChildItem "D:\BeijiXing-AI\frontend\web-pc\src" -Recurse -Include *.js,*.vue).Count
Write-Host "web-admin源码文件: $webAdminSrc"
Write-Host "web-pc源码文件: $webPcSrc"
```

**影响**: 
- 如果确实包含node_modules → **无影响** (正常现象)
- 如果是源码文件过多 → **需要优化** (考虑拆分或重构)

**建议**: 排除node_modules重新统计，更新文档中的准确数字
